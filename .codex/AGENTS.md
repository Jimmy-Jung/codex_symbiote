# .codex 스코프 지침

> Author: jimmy
> Last Updated: 2026-03-13

이 파일은 `.codex/` 디렉터리 작업에만 적용됩니다.
루트 `AGENTS.md`의 공통 규칙 위에 설정 파일 관리 규칙만 추가합니다.

## 1) 디렉터리 책임

- `config.toml`: 프로젝트 스코프 Codex 설정
- `agents/*.toml`: 선택형 멀티에이전트 역할 정의
- `rules/*.rules`: 실행 정책
- `docs/*`: 운영 메모와 참고 문서
- `skills/*`: 필요 시 사용하는 워크플로우

## 2) config.toml 원칙

- 공식 키만 사용합니다.
- 프로젝트 기본값은 최소한으로 유지합니다.
- 기본 템플릿에는 핵심 역할과 최소 스킬만 등록합니다.
- 프로젝트별 확장은 실제 필요가 생길 때 추가합니다.

## 3) agent role 원칙

- 역할은 단일 책임으로 설계합니다.
- 기본 템플릿은 `explorer`, `worker`, `reviewer`, `monitor` 중심으로 유지합니다.
- 분석가, 설계자, QA, 보안 등 세부 역할은 확장용으로 간주합니다.
- 공식 자료 기반 전문가 역할도 `.codex/agents/extensions/`에만 추가하고 기본 `config.toml`에는 등록하지 않습니다.
- 역할 지시는 중복을 줄이고 실제 도구 사용 방식에 맞게 유지합니다.

## 4) 문서 동기화

다음 항목은 서로 맞춰 둡니다.

- 루트 `AGENTS.md`
- `.codex/config.toml`
- `.codex/docs/*`의 운영 문서
- 공식 자료 기반 역할 출처 문서
- Synapse_CoR 기본 출력 규칙

Synapse_CoR 출력 규칙 동기화 기준:
- 출력 시작은 placeholder가 아닌 실제 이모지를 사용합니다.
- 역할 기반 이모지 선택을 기본으로 합니다.
- 같은 응답 안에서는 선택한 이모지를 일관되게 유지합니다.

## 5) 검증 원칙

- 설정 변경 후에는 문서와 설정 간 충돌이 없는지 확인합니다.
- 오래된 명령, 현재 지원되지 않는 키, 실존하지 않는 전제 파일은 기본 규칙으로 강제하지 않습니다.

## 6) 주의

- `.codex/project/context.md`나 `manifest.json` 같은 커스텀 파일은 선택형 컨벤션입니다.
- 기본 템플릿은 특정 내부 워크플로우를 모든 프로젝트에 강제하지 않습니다.
